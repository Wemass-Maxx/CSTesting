WMS HTML Converter

Esta herramienta convierte un mockup HTML con clases wms-* en los archivos necesarios para una plantilla del sistema: tag.js, metadata.json, config.json, definition.json y, si se aporta CSS, outer.css/outer-mobile.css parametrizados.

Archivos de entrada

  • HTML desktop: obligatorio. Debe contener los nodos WMS que el renderer creara.
  • HTML mobile: opcional. Si se activa Desktop + Mobile y no se sube, se reutiliza el desktop.
  • outer.css: opcional. Solo se parametrizan variables CSS marcadas con opt-in.
  • outer-mobile.css: opcional. Usa las mismas reglas que desktop, pero genera campos con DEVICE: "mobile".

Regla principal

El converter no parametriza CSS automaticamente. Una variable CSS solo se convierte en campo de formulario si esta marcada con wms-param. Esto evita tocar variables internas del layout o del renderer.

Marcado por linea

:root {
  --wm-scoreboard-width: 450px; /* wms-param label="Ancho marcador" */
  --wm-scoreboard-scale: 1; /* wms-param type=Decimal label="Escala marcador" */
  --wm-scoreboard-bg-color: #090d14; /* wms-param label="Color fondo marcador" */
}

Marcado por bloque

:root {
  /* wms-param:start section=scoreboard */
  --wm-scoreboard-text-color: #ffffff;
  --wm-scoreboard-radius: 10px;
  --wm-scoreboard-status-bg: #d9ff3f;
  /* wms-param:end */
}

Opciones soportadas

  • field=nombreCampo: fuerza el nombre del campo en definition.json.
  • label="Texto visible": define el texto del formulario.
  • type=String|Text|Enum|Boolean|Integer|Decimal|Image|Video|Zip: fuerza el tipo.
  • device=desktop|mobile|both: fuerza el dispositivo.
  • required=true: marca el campo como obligatorio.
  • section=basic: permite agruparlo con los campos basicos.
  • values=a|b|c: valores para campos Enum.

Assets en CSS

Los assets dentro de CSS son sustituibles solo si la variable esta marcada. Un url(...) marcado se infiere como Image salvo que se indique otro tipo. 

:root {
  --wm-scoreboard-home-flag-image: url("images/flags/BRA.png"); /* wms-param type=Image label="Bandera local" */
}

Salida esperada:

--wm-scoreboard-home-flag-image: url("##scoreboardHomeFlagImage##");

Inferencia de tipos

  • 450px genera Integer y conserva la unidad en CSS.
  • 0.45 genera Decimal.
  • "Texto" genera String y conserva las comillas en CSS.
  • linear-gradient(...), 1px solid #fff o calc(...) generan String.
  • url(...) genera Image cuando esta dentro del opt-in.

Campos internos que conviene no marcar

No marques variables de infraestructura salvo que sepas que deben quedar editables en Creative Studio: --wms-content-width, --wms-skin-width, --wms-skin-height, --wms-stickywrapper-height, --wms-header-height, --wms-safezone-* y variables calculadas por comportamientos del renderer.

Los placeholders CSS usan el contrato del backend: ##campo## y /*##campo##*/. Creative Studio reemplaza esos valores cuando crea la creatividad final.
